# 0.1.29 migration notes

## What changed

Internal SSOT cleanup + legacy alias surface removed. `_BUILTIN_KINDS` is now
derived from module reflection (single-source). Three invariant test suites
added (entry_point resolvability, event-kind constant SSOT, transient-error
classification SSOT). The legacy `rate_limit_*` event kinds, API exports, and
`runtime.rate_limit_action` config field that have been dual-emitting since
0.1.23 are deleted; only the canonical `transient_error_*` surface remains.

## Event consumer migration

Replace old event subscriptions with canonical equivalents:

```
# rate_limit_rejected → transient_error_detected
# (add filter for classification == "rate_limit_account" if you only want
#  the 5h-quota case; other transient errors now go through the same path)
s/rate_limit_rejected/transient_error_detected/

s/rate_limit_recovered/transient_error_recovered/

s/rate_limit_backoff_capped/transient_error_backoff_capped/
```

`transient_error_detected` payload uses `classification` (not `limit_type`):
- `rate_limit_account` — 5h OAuth quota (same case as old `rate_limit_rejected`)
- `rate_limit_model` — 429 without five_hour rate_limit_event
- `api_transient_5xx` — 5xx server errors
- `api_timeout` — 408 timeout

## Config migration

In your `agent-runner.toml`:

```
s/rate_limit_action/transient_error_action/
```

Allowed values are identical: `back_off` / `skip` / `stop`.

If the alias remains in TOML, `load_config` raises `ConfigError` immediately
with a one-line hint pointing to this file.

## API consumer migration

In Python imports:

```python
# before
from agent_runner.api import emit_rate_limit_rejected
from agent_runner.api import emit_rate_limit_recovered
from agent_runner.api import emit_rate_limit_backoff_capped

# after
from agent_runner.api import emit_transient_error_detected
from agent_runner.api import emit_transient_error_recovered
from agent_runner.api import emit_transient_error_backoff_capped
```

## Failure modes if you miss a step

- **Stale config TOML** → `ConfigError: runtime.rate_limit_action was removed in 0.1.29;
  use runtime.transient_error_action ...` on first start.
- **Stale event subscriber** → events silently absent; grep your code for legacy names.
- **Stale Python import** → `ImportError: cannot import name 'emit_rate_limit_rejected'`.

## Why now

The alias surface was introduced in 0.1.20 (`rate_limit_rejected`), generalized
in 0.1.23 (`transient_error_detected` as canonical), and dual-emitting since.
Downstream consumers confirmed test-mode integration (not production workloads);
cutting the aliases now avoids carrying back-compat dual-emission indefinitely.

## Verification recipe

Run in your consumer repo before upgrading:

```bash
grep -rnE 'rate_limit_(rejected|recovered|backoff_capped|action)|emit_rate_limit_' . \
    && echo "found references — migrate" || echo "clean"
```

If clean, upgrade is zero-work. If output found, apply s/// patches above.
