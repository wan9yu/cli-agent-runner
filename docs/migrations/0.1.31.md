# Migrating to 0.1.31

## What changed

0.1.31 ships three additive defenses and two doc clarifications. All new
features default OFF — no config migration is required. Existing deployments
are unaffected unless you explicitly opt in by setting the new config keys.

## HUNG: `max_grace_after_result_s`

A subprocess that emits `{"type":"result"}` then sits silent is a HUNG agent.
0.1.31 adds a kill countdown that fires when this condition is detected.

**Enable** (recommended starting point for short-prompted agents):

```toml
[runtime]
max_grace_after_result_s = 30
```

Long-running agents (multi-hour gate agents) may need a higher value:

```toml
[runtime]
max_grace_after_result_s = 1200
```

**Event emitted**: `round_grace_kill` (payload: `round_num`, `grace_s`).
Distinct from `round_timeout_kill` — postmortem can distinguish grace-kill
(result seen, subprocess still running) from wall-clock timeout (no result).

**Default**: `0` (disabled). Set to the longest expected "result to exit"
window for your agent. The grace countdown begins the moment `"type":"result"`
appears in the agent's JSONL stdout log.

## Repetitive-tool anomaly: `anomaly_repetitive_window` + `anomaly_repetitive_threshold`

A stuck "Edit → Edit → Edit → ..." loop is a common agent failure mode.
0.1.31 adds a sliding-window detector in the claude plugin.

**Enable** (conservative starting point):

```toml
[monitor]
anomaly_repetitive_window = 10
anomaly_repetitive_threshold = 8
```

Tune per project by observing `anomaly_repetitive_tool` events in dev runs.
Lower threshold = more sensitive; higher = fewer false positives.

**Event emitted**: `anomaly_repetitive_tool` (payload: `round_num`,
`tool_name`, `target`, `count`, `window`).

**Scope**: claude only. Gemini's JSONL exposes per-tool stats summaries, not
per-event tool calls; a separate detector path would be needed for parity.

**Default**: both `0` (disabled). Both must be `> 0` to activate.

## `dry_run` flag passthrough

Plugin authors doing multi-stage rollout or side-effect isolation can now
read `ctx.dry_run` to skip irreversible actions:

```toml
[runtime]
dry_run = true
```

```python
def after_round(self, ctx: HookContext, result: Any) -> None:
    if ctx.dry_run:
        return  # skip git push, webhook call, etc.
    # ... normal side-effect code ...
```

Built-in plugins (`claude_error_detector`, `gemini_error_detector`) have no
side effects and ignore the flag.

**Default**: `false`.

## Doc clarifications (no action required)

- **Config reload**: `docs/configuration.md` now documents that TOML changes
  require `agent-runner restart`. The supervisor reads config once at startup
  and reuses it across all rounds in a serve session.
- **Phase rotation indexing**: `docs/configuration.md` `[phases]` section now
  documents the `round_num % len(phases)` formula so operators resuming from
  a non-zero round counter understand the apparent "phase offset".
