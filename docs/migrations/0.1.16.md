# 0.1.16 Migration recipes

## Breaking: `runtime.round_timeout_per_phase` dict syntax removed

The dict syntax has been replaced with `[phases.<name>]` sub-tables. The new
schema accommodates any per-phase field (including `prompt.files` shipped in
the same release) without N² growth.

### Before (0.1.15 and earlier)

```toml
[phases]
list = ["dev", "qa", "product"]

[runtime]
round_timeout_per_phase = { dev = 3600, qa = 900, product = 600 }
```

### After (0.1.16+)

```toml
[phases]
list = ["dev", "qa", "product"]

[phases.dev]
round_timeout_s = 3600

[phases.qa]
round_timeout_s = 900

[phases.product]
round_timeout_s = 600
```

## Breaking: `Config.phases` type change

`Config.phases` was `list[str] | None`; it's now a `PhasesConfig` dataclass
with `.list` and `.overrides` fields. If your code or plugin reads `cfg.phases`
as a list directly:

```python
# Before
for phase in cfg.phases:  # iterates raw list
    ...

# After
for phase in cfg.phases.list or []:  # iterates the .list field
    ...
```

```python
# Before
assert cfg.phases == ["dev", "qa"]

# After
assert cfg.phases.list == ["dev", "qa"]
```

## New: Multi-file prompt concat

To concat multiple files (e.g. preamble + role-specific body), use
`prompt.files`:

```toml
[prompt]
files = ["_common.md", "dev.md"]
```

Default separator is `"\n\n"` (markdown-safe). Override to `"\n\n---\n\n"` for
visible section breaks:

```toml
[prompt]
files = ["_common.md", "dev.md"]
concat_separator = "\n\n---\n\n"
```

Per-phase override:

```toml
[phases.qa]
prompt.files = ["_common.md", "qa.md"]
```

Single-file `prompt.file = "x.md"` continues to work unchanged.

## New: Frontmatter strip opt-out

YAML frontmatter on the first prompt file is stripped by default (R721 defense
against `claude -p '---...'` arg-parse rejection). To preserve frontmatter for
non-LLM-CLI agents:

```toml
[prompt]
files = ["a.md"]
strip_yaml_frontmatter = false
```
