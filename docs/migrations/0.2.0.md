# Migration guide: 0.2.0

## TL;DR — zero migration for existing users

`[vcs] dirty_action = "stash" | "ignore" | "auto_commit"` works exactly as
before. No TOML changes, no behavioral changes. The implementation moved from
core into the bundled `default_dirty_handler` plugin (default-on), but that is
transparent to operators who are not customising dirty-tree handling.

---

## What changed internally

Dirty-tree handling after a clean-exit round is now routed through a priority-
ordered chain of `DirtyHandler` plugins rather than a hardcoded switch inside
core. The first handler to return a non-`None` `DirtyOutcome` wins; remaining
handlers are skipped.

The bundled `default_dirty_handler` plugin (priority 1000) ships enabled and
reproduces the pre-0.2.0 behavior exactly:

| `[vcs] dirty_action` | Outcome |
|---|---|
| `"stash"` (default) | Stash with ORPHAN-prefix message. `dirty_detected` + `orphan_stashed` events emitted. |
| `"ignore"` | Emit `dirty_detected` only. Tree left dirty for next round. |
| `"auto_commit"` | Commit with subject `agent-runner auto-commit: R<N> <phase>`. `dirty_auto_committed` event emitted on success. On failure, emits `dirty_commit_failed` and leaves tree dirty. |

The stash *mechanism* (git primitives, SHA lock, idempotency guard) remains in
core unchanged. Only the policy — which action to take and when — is now
plugin-provided.

---

## New: `RoundResult.dirty_outcome`

`RoundResult` gains `dirty_outcome: DirtyOutcome | None = None` (additive).
`PostRoundHook` authors can read it:

```python
from agent_runner.api_types import DirtyOutcome

class MyPostRoundHook:
    name = "my_hook"

    def after_round(self, ctx, result):
        outcome: DirtyOutcome | None = result.dirty_outcome
        if outcome and outcome.kind == "committed":
            # handler auto-committed; outcome.ref is the commit SHA
            ...
```

---

## New events

| Event | When |
|---|---|
| `dirty_auto_committed` | A handler (including the bundled default) successfully auto-committed the dirty tree. Payload: `round_num`, `files`, `ref` (commit SHA). |

Existing events (`dirty_detected`, `orphan_stashed`, `orphan_stash_failed`,
`dirty_commit_failed`) are unchanged.

---

## How to own your dirty-tree policy (consumer plugin recipe)

To replace the bundled behavior entirely:

**Step 1 — disable the bundled default:**

```toml
# agent-runner.toml
[plugins]
disable = ["default_dirty_handler"]
```

**Step 2 — register your own handler on `agent_runner.dirty_handler_hooks`:**

```toml
# your_plugin/pyproject.toml
[project.entry-points."agent_runner.dirty_handler_hooks"]
my_dirty_handler = "your_plugin.dirty:MyDirtyHandler"
```

```python
# your_plugin/dirty.py
from agent_runner import api
from agent_runner.api_types import DirtyOutcome
from agent_runner.hooks import HookContext, register_dirty_handler


class MyDirtyHandler:
    name = "my_dirty_handler"
    priority = 10  # ascending; only matters when multiple handlers coexist

    def handle_dirty(
        self,
        ctx: HookContext,
        dirty_files: list[str],
        result,
    ) -> DirtyOutcome | None:
        # Return None to pass to the next handler in the chain.
        # Return a DirtyOutcome to claim the result and stop dispatch.
        # Example policy: commit doc-only churn, stash everything else.
        if all(f.endswith(".md") for f in dirty_files):
            sha = api.try_auto_commit(
                ctx.work_dir, ctx.round_num, ctx.phase, log_dir=ctx.log_dir
            )
            return DirtyOutcome(kind="committed", ref=sha) if sha else DirtyOutcome(kind="ignored")
        ref = api.stash_orphan(
            ctx.work_dir,
            round_num=ctx.round_num,
            phase=ctx.phase,
            idempotency_s=ctx.vcs.stash_idempotency_s if ctx.vcs else 5,
            log_dir=ctx.log_dir,
        )
        if ref is None:
            return DirtyOutcome(kind="ignored")
        return DirtyOutcome(kind="stashed", ref=ref.sha)


# Module-top side effect — fires at entry_point load time.
register_dirty_handler(MyDirtyHandler())
```

**`DirtyOutcome` values:**

| `kind` | Meaning | `ref` |
|---|---|---|
| `"stashed"` | Tree was stashed | stash SHA |
| `"committed"` | Tree was committed | commit SHA |
| `"ignored"` | Tree left dirty intentionally | `None` |

**Calling core primitives from a handler:**

```python
from agent_runner import api

# Stash dirty tree (returns a StashRef (with .sha) or None if nothing to stash)
ref = api.stash_orphan(
    ctx.work_dir,
    round_num=ctx.round_num,
    phase=ctx.phase,
    idempotency_s=ctx.vcs.stash_idempotency_s if ctx.vcs else 5,
    log_dir=ctx.log_dir,
)
outcome = DirtyOutcome(kind="ignored") if ref is None else DirtyOutcome(kind="stashed", ref=ref.sha)

# Auto-commit dirty tree (returns commit SHA or empty string; raises AutoCommitError on failure)
sha = api.try_auto_commit(
    ctx.work_dir,
    ctx.round_num,
    ctx.phase,
    log_dir=ctx.log_dir,
)
```

## `peek --select defenses` note

The stash *mechanism* entries in the defenses catalog (`sha_locked_stash`,
`orphan_stash_idempotency_s`) remain in core unchanged. `peek --select defenses`
continues to show them as active when applicable. The *policy* — which action to
apply on a dirty tree — is now sourced from the bundled `default_dirty_handler`
plugin rather than a hardcoded core switch.
