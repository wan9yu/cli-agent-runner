# Migrating to 0.1.30

## What changed

0.1.30 fixes a P0 bug where `ClaudeErrorDetector` and `GeminiErrorDetector`
silently dropped all events when the user set a custom `[agent] name` in their
TOML config. Both plugins now guard on `HookContext.agent_binary` (the
basename of `cfg.agent.command[0]`) instead of `HookContext.agent_name` (the
user-cosmetic name field). Three infrastructure additions ship alongside the
bug fix: upstream-CLI fixture canary tests, a Layer 2 LOC invariant, and
`docs/thesis.md` as project non-goals SSOT.

No event payload schemas changed. No public Python API changed. No CLI
surface changed.

## For consumers (agent-runner.toml users)

**No migration action required.**

If your config has a custom `[agent] name`:

```toml
[agent]
command = ["claude", "--some-flag"]
name = "acme_dev"   # custom cosmetic name
```

Previously this silenced all `agent_usage_recorded` and
`transient_error_detected` events. After 0.1.30 those events fire correctly
based on the command binary (`claude`), not the name.

If you previously worked around the bug by setting `name = "claude"` (or
`name = "gemini"`), your workaround is now unnecessary — you can use any name.

## For plugin authors

`HookContext` has a new optional field:

```python
@dataclass(frozen=True)
class HookContext:
    agent_name: str | None     # user-cosmetic [agent] name field
    agent_binary: str | None   # basename of cfg.agent.command[0]  (NEW 0.1.30)
    ...
```

**Prefer `agent_binary` for capability detection.** `agent_name` is cosmetic
and set by the operator; `agent_binary` is the actual command invoked.

Example guard pattern:

```python
def after_round(self, ctx: HookContext, result: Any) -> None:
    if ctx.agent_binary != "claude":   # use agent_binary, not agent_name
        return
```

`agent_binary` defaults to `None` for `HookContext` instances constructed
without it (rare; tests use `make_hook_context` which defaults `agent_binary`
to `agent_name` for back-compat). Guard against `None` if your plugin runs in
contexts where the field may not be populated:

```python
if not ctx.agent_binary or ctx.agent_binary != "claude":
    return
```

## Thesis doc

See [`docs/thesis.md`](../thesis.md) for the project non-goals SSOT. Future
feature requests are evaluated against it. Accepted: specific observed failure
modes, generic across agent types, no wrong-default knobs. Rejected: BI / analytics,
novelty detection, multi-tenant, prompt-engineering frameworks, remediation chains.
