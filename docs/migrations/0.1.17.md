# 0.1.17 Migration recipes

## Deprecation: `vcs.orphan_action` → `vcs.dirty_action`

Old name still works in 0.1.17 (DeprecationWarning to stderr at config
load). Removed in 0.1.18.

### Before

```toml
[vcs]
orphan_action = "stash"
```

### After

```toml
[vcs]
dirty_action = "stash"
```

## New: choose `dirty_action` per project's git ownership model

If your project's agent owns git (commits + pushes during round body):

```toml
[vcs]
dirty_action = "ignore"   # supervisor doesn't touch dirty tree
```

If you want supervisor to auto-commit dirty changes (no push):

```toml
[vcs]
dirty_action = "auto_commit"
```

Auto-commit subject is hardcoded:
`agent-runner auto-commit: R<N> <phase>`. No body, no AI attribution.

If commit fails (e.g. missing `git config user.email`), supervisor
emits `dirty_commit_failed` event and falls back to leaving tree dirty.

## Better: relative paths now work

In 0.1.16, `prompt.files = ["prompts/a.md"]` only worked when CWD ==
work_dir. 0.1.17 resolves all relative paths against `runtime.work_dir`
at config load. You can simplify absolute-path workarounds:

### Before (0.1.16 workaround)

```toml
[prompt]
files = [
    "/home/user/myproject/prompts/_common.md",
    "/home/user/myproject/prompts/diverge.md",
]
```

### After (0.1.17 native)

```toml
[prompt]
files = ["prompts/_common.md", "prompts/diverge.md"]
```

Both forms work; relative is more portable across deployments.

## New: agent can read its own round_num + phase from env

`agent-runner serve` now injects `AGENT_RUNNER_ROUND_NUM` and
`AGENT_RUNNER_PHASE` into the round subprocess env (alongside existing
`AGENT_RUNNER_LOG_DIR` from 0.1.15). Agents can use these to:

```bash
echo "starting round R$AGENT_RUNNER_ROUND_NUM (phase=$AGENT_RUNNER_PHASE)"
```

`AGENT_RUNNER_PHASE` is `""` (empty string) when no `[phases]` section is
configured.
