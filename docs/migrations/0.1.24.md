# 0.1.24 migration notes

## What changed

### 1. New `agent_usage_recorded` event

Each round (claude or gemini) now emits an `agent_usage_recorded` event
post-round with token + cost data:

```jsonc
{
  "event": "agent_usage_recorded",
  "agent": "claude",          // or "gemini"
  "model": "claude-opus-4-7", // or "gemini-3-flash-preview" | ...
  "round_num": 42,
  "input_tokens": 20,
  "output_tokens": 6,
  "cached_tokens": 80,
  "cost_usd": 0.0812,         // claude = USD float; gemini = null
  "duration_ms": 14470,
  "models_breakdown": null    // gemini multi-model rounds only; null otherwise
}
```

**This is raw data only.** Aggregation (24h totals, per-model breakdowns,
tier-aware budget warnings) is deferred to consumers and the 0.1.25
capability layer. Operators wanting "how much have I spent today" can
write a script that sums the events over their desired time window.

**Field semantics:**

- `input_tokens`: net non-cached input. Both claude and gemini emit this
  as the "fresh tokens billed at full rate" — for total throughput add
  `input_tokens + cached_tokens`.
- `cached_tokens`: tokens served from prompt cache (claude) or context
  cache (gemini). Generally cheaper or free depending on tier.

### 2. New gemini built-in plugin

`agent_runner/builtin_plugins/gemini.py` ships with the package and
auto-registers as a `post_round_hook`. For claude-only deployments, the
plugin is a no-op (returns early when `ctx.agent_name != "gemini"`). For
gemini deployments, it emits:

- `agent_usage_recorded` from `result.stats` after every round
- `transient_error_detected` for gemini 5xx / 429 / 408 errors (best-effort
  classifier; gemini error JSONL schema is less stable than claude's, so
  unknown error codes silently skip)

Operators can disable via `[plugins] disable = ["gemini_error_detector"]`.

### 3. Gemini preset `--skip-trust` hotfix

The 0.1.19 `gemini` preset shipped without `--skip-trust`, which made
unattended operation fail in any directory not previously trusted via
the gemini CLI's interactive trust prompt. Headless agent-runner usage
is the primary use case, so the flag is now in the preset.

```diff
 [agent]
-command = ["gemini", "--yolo", "-o", "text"]
+command = ["gemini", "--yolo", "--skip-trust", "-o", "text"]
```

**Existing scaffolds (TOML files generated by 0.1.19–0.1.23
`agent-runner init --preset gemini`) need a manual edit** to add the
flag. New scaffolds from 0.1.24+ ship the fix automatically.

Same semantic as claude's `--dangerously-skip-permissions` flag — both
CLIs require explicit "I trust this directory" opt-in for unattended
operation.

## Migration

None required for new deployments. Existing gemini scaffolds need a
1-line TOML edit (see Section 3 above). Plugins/scripts subscribed to
`transient_error_*` events automatically receive gemini errors too —
no subscriber change needed.

## What's NOT in this release

- **Usage aggregation views** (e.g. 24h totals, per-model breakdowns,
  cost projections). Deferred to 0.1.25.
- **Tier-aware budget warnings**. Deferred to 0.1.25.
- **HTTP monitor usage panel**. Deferred to 0.1.25.
- **`agent-runner usage` exporter**. Deferred to 0.1.25.
- **`cache_creation_tokens` claude-specific field**. Deferred to 0.1.25
  if aggregation needs it.
- **Aider plugin**. Not tested in this release.

## How to consume usage events (sketch for plugin authors)

```python
# Reading events.jsonl to compute 24h cost (claude only)
import json
import time
from pathlib import Path


def compute_24h_cost(log_dir: Path) -> float:
    cutoff = time.time() - 86400
    total = 0.0
    for events_file in log_dir.glob("events-*.jsonl"):
        for line in events_file.read_text().splitlines():
            if not line.strip():
                continue
            ev = json.loads(line)
            if ev.get("event") != "agent_usage_recorded":
                continue
            if ev.get("agent") != "claude":  # gemini has cost_usd=null
                continue
            total += ev.get("cost_usd") or 0
    return total
```

A 0.1.25 helper `agent_runner.api.compute_recent_usage(log_dir, since_s)`
will offer a sanctioned version of this pattern.
